# 科技企业知识助手 - 功能需求文档

## 一、项目概述

### 1.1 产品名称
科技企业知识助手（Tech Enterprise Knowledge Assistant）

### 1.2 产品定位
面向科技企业研发团队的智能知识管理平台与对话机器人，以"AI 破解技术知识孤岛、数据激活研发效率"为核心，打通技术文档、代码库、API文档、产品知识库等异构数据壁垒，实现知识检索、智能问答、代码生成、任务执行的统一入口。

平台整合企业技术文档（API文档、SDK手册、架构设计文档）、代码仓库（GitHub/GitLab）、产品知识库（产品说明书、规格参数）、开发流程（CI/CD、代码评审）等多源信息，依托大语言模型、检索增强生成、智能体等核心能力，为研发人员提供 7×24 小时不间断的智能技术服务，同时为技术管理团队减负增效，实现知识利用率提升与研发成本优化的双重目标。

### 1.3 目标用户
- **技术负责人/CTO**：管理技术知识库、监控团队研发效率、进行技术决策支持
- **研发团队负责人**：管理团队知识库、协调技术攻关、指导团队成员
- **开发工程师**：查询技术文档、获取代码示例、解决开发问题
- **测试工程师**：查询测试文档、获取测试用例、了解产品功能

### 1.4 技术栈
- **后端**：FastAPI + SQLite + OpenSearch（向量存储）+ Redis（对话历史/状态管理）+ MinerU（文档解析）+ Ollama（嵌入）+ DeepSeek（LLM）+ SearXNG（联网搜索）
- **前端**：Vue 3 + Vite + Element Plus + Pinia + Axios
- **智能体框架**：LangGraph（高级智能体编排）

### 1.5 核心价值主张
1. **真实需求导向**：解决研发人员"找文档难、问问题难、学新技术难"的痛点
2. **AI价值突出**：没有AI做不好的场景——智能问答、代码生成、技术方案设计
3. **商业前景明确**：B端付费能力强，可按团队规模订阅
4. **数据可获得性**：技术文档、代码库都是企业内部已有数据

## 二、功能需求清单

### 模块一：技术知识智能问答（RAG核心）

| 功能点 | 详细需求 | 优先级 |
|--------|----------|--------|
| API文档问答 | 支持自然语言查询API用法（如"如何调用用户登录接口？""订单接口的参数有哪些？"），基于RAG从API文档精准检索，生成代码示例和参数说明 | 高 |
| SDK手册问答 | 查询SDK使用方法（如"如何初始化SDK？""文件上传的SDK方法是什么？"），提供代码示例和注意事项 | 高 |
| 架构设计问答 | 查询系统架构设计（如"用户服务的架构是什么？""数据流向是怎样的？"），返回架构图描述和关键设计决策 | 高 |
| 多轮对话与追问 | 具备多轮对话能力，维持上下文记忆。当需求模糊时主动追问（如"请问您使用的是Java还是Python SDK？""您是哪个版本的API？"） | 高 |
| 技术对比推荐 | 支持就多个技术方案进行对比（如"Redis和Memcached怎么选？""REST和GraphQL的区别？"），从性能、易用性、适用场景等维度对比 | 中 |
| 场景化技术引导 | 面向典型开发场景的技术引导（如"如何实现用户认证？""如何设计高并发接口？""如何进行单元测试？"），推荐匹配的技术方案和代码示例 | 中 |
| 代码示例生成 | 根据自然语言描述生成代码示例（如"写一个Python函数发送HTTP请求""写一个Vue组件展示表格数据"），结合知识库中的代码规范 | 高 |
| 答案溯源与引用 | 标注信息来源（如"来自《API文档v2.0》第15页""来自GitHub仓库src/user/service.py"），支持一键跳转查看原文 | 高 |
| 兜底应答机制 | 当知识库缺失相关信息时，如实告知并引导查阅官方文档或联系技术负责人，严禁编造答案 | 高 |

### 模块二：智能体工具（Function Calling）

| 功能点 | 详细需求 | 优先级 |
|--------|----------|--------|
| 代码搜索 | 搜索代码仓库中的代码片段（如"查找用户认证相关的代码""搜索订单创建的实现"），返回代码位置和上下文 | 高 |
| CI/CD状态查询 | 查询CI/CD流水线状态（如"上次构建失败的原因是什么？""部署进度如何？"），返回构建日志和状态信息 | 中 |
| 代码评审辅助 | 辅助代码评审（如"这段代码有没有潜在问题？""有没有安全隐患？""是否符合代码规范？"），AI分析代码并给出改进建议 | 中 |
| Bug分析诊断 | 输入错误日志或异常堆栈，AI分析可能的原因并给出修复建议（如"NullPointerException怎么解决？""数据库连接超时怎么办？"） | 高 |
| API调用测试 | 自然语言描述API调用，AI生成测试代码并执行（如"测试用户登录接口，传入username和password"），返回测试结果 | 中 |
| Pandas数据分析 | 上传CSV/Excel，用自然语言进行数据分析（如"按模块统计代码行数""统计接口响应时间分布"），生成表格和图表 | 中 |

### 模块三：高级智能体编排（LangGraph）

| 功能点 | 详细需求 | 优先级 |
|--------|----------|--------|
| 技术方案设计 | 用户描述业务需求（如"设计一个电商购物车系统"），智能体进行任务拆解：1) 需求分析 2) 架构设计 3) 数据库设计 4) API设计 5) 代码实现建议，每一步有状态管理和记忆 | 高 |
| 代码重构建议 | 用户输入代码片段，智能体进行多步分析：1) 识别问题 2) 提出重构方案 3) 生成重构后的代码 4) 验证重构结果，支持自我修正 | 中 |
| 技术文档生成 | 用户提供代码或功能描述，智能体自动生成技术文档：1) 分析代码结构 2) 提取关键信息 3) 按照模板生成文档 4) 优化文档内容 | 中 |
| 多任务协同处理 | 处理有依赖关系的复杂任务（如"先查询API文档，然后生成调用代码，最后编写单元测试"），智能体管理任务依赖和执行顺序 | 中 |

| 知识库删除 | 支持逻辑删除，删除前校验是否有引用 | 中 |

| 内容质量评估 | 统计内容使用情况（被检索次数、被引用次数），识别高价值内容与低质量内容，为内容运营提供依据 | 低 |

## 三、技术架构

### 3.1 系统架构图

```
┌─────────────────────────────────────────────────────────────┐
│                      前端交互层                              │
│  Vue 3 + Element Plus + 对话式界面 + 流式响应 + 富媒体展示   │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP/WebSocket
┌──────────────────────────▼──────────────────────────────────┐
│                      后端服务层                              │
│  FastAPI + SQLite + JWT + Redis + 业务逻辑 + API路由         │
└──────────────────────────┬──────────────────────────────────┘
                           │ API调用
┌──────────────────────────▼──────────────────────────────────┐
│                      AI能力层                                │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│  │    RAG链     │  │   智能体工具  │  │  LangGraph   │       │
│  │ 检索→重排→   │  │ 代码搜索/CI/ │  │ 高级智能体   │       │
│  │ 生成→溯源    │  │ CD/评审/诊断 │  │ 状态管理/    │       │
│  │              │  │              │  │ 多步推理     │       │
│  └──────────────┘  └──────────────┘  └──────────────┘       │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│  │ Embedding    │  │   LLM调用    │  │ 向量检索     │       │
│  │ Ollama       │  │ DeepSeek     │  │ OpenSearch   │       │
│  └──────────────┘  └──────────────┘  └──────────────┘       │
└──────────────────────────┬──────────────────────────────────┘
                           │ 数据接入
┌──────────────────────────▼──────────────────────────────────┐
│                      数据层                                  │
│  PDF/DOCX/MD文档 + GitHub代码库 + 网页 + CSV结构化数据       │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 RAG链路

```
用户提问 → 查询重写(LLM) → 混合检索(OpenSearch语义+关键词) 
→ 结果重排(LLM) → 答案生成(LLM) → 来源溯源 → 流式返回
```

### 3.3 智能体编排（LangGraph）

```
状态: {task, context, step, result, history}

开始 → 需求分析 → 方案设计 → 代码生成 → 验证测试 → 结束
         ↓            ↓            ↓            ↓
      检索知识     调用工具     生成代码     执行测试
```

## 四、页面原型描述

### 4.1 登录页
- 用户名/密码输入框
- 登录按钮
- 记住我复选框

### 4.2 主布局（侧边栏+顶栏）
- **左侧边栏**：仪表盘、知识库管理、文档管理、对话机器人、用户管理、系统配置、对话日志
- **顶部栏**：用户头像、退出登录、系统名称

### 4.3 仪表盘（Dashboard）
- 统计卡片：知识库数量、文档数量、今日咨询数、活跃开发者数
- 热门技术问题排行
- 知识库使用统计图表
- 快捷操作：创建知识库、上传文档、开始对话

### 4.4 对话机器人页（核心交互）
- **左侧**：对话历史列表
- **中间**：消息列表（用户消息、AI回复、代码块高亮、答案引用）
- **右侧**：知识库选择器、智能体工具入口、对话配置
- **底部**：输入框、发送按钮、快捷提问模板

### 4.5 知识库列表页
- 知识库卡片列表（名称、描述、文档数、创建时间、负责人、技术栈标签）
- 筛选栏（按名称、团队、技术栈）

### 4.6 文档列表页
- 文档表格（文件名、大小、上传时间、状态、所属知识库、技术栈）
- 上传按钮（支持拖拽）
- 删除/预览/编辑操作

### 4.7 用户管理页
- 用户表格（用户名、邮箱、团队、角色、状态）
- 创建/编辑/删除用户弹窗

### 4.8 系统配置页
- 模型配置（Embedding模型、LLM模型）
- 文档解析配置（chunk_size、上传限制）
- 检索参数（top_k、相似度阈值）
- 对话配置（上下文轮数、回答长度）

## 十、功能亮点

### 10.1 AI核心能力突出
- 完整的RAG链路，确保AI答复"有据可依、可追溯"
- 智能体工具调用，从"回答问题"升级到"解决问题"
- LangGraph高级编排，支持复杂多步推理和状态管理

### 10.2 科技企业场景专精
- 聚焦研发团队需求：API文档问答、代码生成、Bug诊断
- 支持代码仓库接入，自动解析README和代码注释
- 面向技术方案设计、代码重构等复杂任务

### 10.3 交互体验优秀
- 流式响应，实时展示AI思考过程
- 代码块语法高亮，支持多种编程语言
- 答案引用标注，支持一键跳转查看原文

### 10.4 数据驱动持续优化
- 咨询热点分析，识别技术趋势
- 问答质量评估，持续提升系统能力
- 知识盲区识别，主动提示补充内容

### 10.5 灵活的智能体扩展
- 通过tools.yaml配置文件注册自定义工具
- LangGraph图编排，支持灵活的流程定义
- 无需修改核心代码即可扩展新功能
